Formularios DOM

Validación de datos

  • Grupo 8
  • Diciembre, 2021

La validación de datos se trata de un proceso que nos permite asegurar que la entrada del usuario sea limpia, correcta y útil.

Tenemos:

  • Validación del lado del servidor
  • Validación del lado del cliente

    • Validación de restricciones HTML

    • Grupo 8
    • Diciembre, 2021

    Se basa en:

  • Atributos de entrada HTML de validación de restricciones
  • Pseudo selectores CSS de validación de restricciones
  • Propiedades y métodos DOM de validación de restricciones
    • Leer más

    Atributos y selectores


      Atributos de entrada HTML de validación de restricciones

    Atributo Descripción
    disabled Especifica que el elemento de entrada debe estar deshabilitado.
    max Especifica el valor máximo de un elemento de entrada.
    min Especifica el valor mínimo de un elemento de entrada.
    pattern Especifica el patrón de valor de un elemento de entrada.
    required Especifica que el campo de entrada requiere un elemento
    type  Especifica el tipo de un elemento de entrada.

      Pseudo selectores CSS de validación de restricciones

    Selector Descripción
    :disabled Selecciona elementos de entrada con el atributo "deshabilitado" especificado
    :invalid Selecciona elementos de entrada con valores no válidos.
    :optional Selecciona elementos de entrada sin ningún atributo "obligatorio" especificado
    :required Selecciona elementos de entrada con el atributo "obligatorio" especificado
    :valid Selecciona elementos de entrada con valores válidos

    Ejemplos

    1. Si un campo de formulario (fname) está vacío, esta función alerta un mensaje y devuelve falso para evitar que se envíe el formulario



    Código
    < !DOCTYPE html>
    < html>
    < head>

    < script>
    function validateForm() {
    let x = document.forms["myForm"]["fname"].value;
    if (x == "") {
    alert("El nombre debe llenarse");
    return false;
    }
    }
    < /script>

    < /head>
    < body>
    < h2>JavaScript Validation< /h2>

    < form name="myForm" action="/action_page.php" onsubmit="return validateForm()" method="post">
    Name: < input type="text" name="fname">
    < input type="submit" value="Enviar">
    < /form>

    < /body>
    < /html>


    Esta es la manera en que podemos visualizar nuestro ejemplo al ejecutarlo



    2. Si un campo de formulario (fname) está vacío, el atributo required evita que se envíe este formulario



    Código
    < !DOCTYPE html>
    < html>
    < body>
    < h2>JavaScript Validation< /h2>

    < form action="/action_page.php" method="post">
    < input type="text" name="fname" required>
    < input type="submit" value="Enviar">
    < /form>

    < p>Si hace clic en enviar, sin completar el campo de texto, su navegador mostrará un mensaje de error.< /p>

    < /body>
    < /html>


    Esta es la manera en que podemos visualizar nuestro ejemplo al ejecutarlo



    3. JavaScript se usa a menudo para validar la entrada numérica



    Código
    < !DOCTYPE html>
    < html>
    < body style="background-color: #E5F4EB;">
    < h2>JavaScript Validation< /h2>
    < p>Porfavor ingresa un número entre el 1 y el 10:< /p>
    < input id="numb">
    < button type="button" onclick="myFunction()">Enviar< /button>
    < p id="demo">< /p>

    < script> function myFunction() {
    let x = document.getElementById("numb").value;
    let text;
    if (isNaN(x) || x < 1 || x > 10) {
    text = "Entrada no válida";
    } else {
    text = "Entrada correcta";
    }
    document.getElementById("demo").innerHTML = text;
    }
    < /script>

    < /body>
    < /html>


    Esta es la manera en que podemos visualizar nuestro ejemplo al ejecutarlo
